# Command Line Utility Reference

| Command Name | Description | Arguments |
|---|---|---|
| `help` | Displays help information for commands. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`command_name` | The specific command to get help for. | str | Yes |
| `action` | Execute the specified action with optional settings as a series of key=value pairs separated by commas | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`action_class` | Class name of the action to execute | str | No<br>`-s, --settings` | Settings for the action as a series of key=value pairs separated by commas | str | Yes |
| `saveas` | Make a copy of the current scene file to the specified location, simulating a Save As... function | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`save_file` | Path to save a copy of the current scene file to | str | No |
| `listproducts` | Dump some basic product metadata for all products in the content management system to the specified output file. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`output_file` | Path to save output to | str | No |
| `load-scene` | Load the specified scene file, replacing the current scene if --merge is not used. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`scene_file` | Path to scene file to load | str | No<br>`-m, --merge` | If specified, merge the new scene into the current scene instead of replacing. | flag (boolean) | Yes |
| `lora-trainer-prep` | Render a matrixed series of images suitable for LORA training. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`scene_files` | A glob pattern that identifies the scene files to load. Each scene file should include a single model which has the hair and clothing applied | str | No<br>`lora_trainer_config` | Path to config file defining the render matrix of shots x poses x expressions | str | No |
| `save-content-item` | Save the selected items in the content library to the specified target location | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`target_dir` | Location to save content item to | str | No |
| `save-subset` | Save the selected items to a scene subset file | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`subset_file` | Location to save content item to | str | No<br>`-d, --directory` | Directory to store the subset file in | str | Yes<br>`-c, --category` | Category to associate subset with | str | Yes |
| `scene-render` | Direct render the current scene, or a new scene file if specified, to the given output file. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`-s, --scene-file` | Name of the scene file to load. | str | Yes<br>`-o, --output-file` | Path to output file to save to | str | Yes |
| `drop-object` | Drop one object on top of another, respecting bounding box boundaries. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`source_node` | Label of the object to be dropped | str | No<br>`target_node` | Label of the object to be dropped onto | str | No |
| `rotate-random` | Rotate and render the selected character. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`object_name` | Label of the object to rotate prior to rendering | str | No<br>`lower` | Starting rotation (in degrees) | int | Yes |
| `rotate-render` | Rotate and render the selected character. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`object_name` | Label of the object to rotate prior to rendering | str | No<br>`lower` | Starting rotation (in degrees) | int | Yes<br>`upper` | Ending rotation (in degrees). rotation-start must be smaller than rotation-end | int | Yes<br>`slices` | How many rotations between start and end to rotate. If set to zero, only render, do not rotate. | int | Yes<br>`-o, --output-file` | Absolute path to directory to render images in. Default is to generate one from the scene file name. | str | Yes<br>`-s, --skip-render` | Flag whether to render or the rotated character or not. Setting this flag skips the rendering step. | flag (boolean) | Yes |
| `transform-copy` | Perform translate, rotate, and/or scale operations on target node to match the transforms on the target node. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`source_node` | Node to translate from | str | No<br>`target_node` | Node to translate to | str | No<br>`-r, --rotate` | Apply the rotation transform | flag (boolean) | Yes<br>`-t, --translate` | Apply the translation transform | flag (boolean) | Yes<br>`-s, --scale` | Apply the scaling transform | flag (boolean) | Yes<br>`-a, --all` | Apply rotation, translation, and scaling trasforms | flag (boolean) | Yes |
| `create-cam` | Create a perspective camera with given name and camera type as parameters to generate the name of the camera along with the selected node label. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`cam_name` | Name of the camera | str | No<br>`cam_class` | Class (type) of camera | str | No<br>`-f, --focus` | If true, turn DOF on the new camera | flag (boolean) | Yes |
| `create-group` | Group the selected nodes in the scene to the name specified. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`group_name` | Name of the new group. | str | No |
| `copy-camera` | Copy the specified camera (by label) to the specified target camera, or the current viewport camera if target not specified | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`-s, --source-camera` | Camera to copy from | str | Yes<br>`-t, --target-camera` | Camera to copy to (optional) | str | Yes |
| `apply-pose` | Apply the given named posed as pose file to the currently selected node or specified node | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`pose_file` | Absolute path to the pose file to be applied | str | No<br>`-t, --target-node` | Label of the node to apply the pose to. If not specified, apply to the currently selected node. | str | Yes |
| `inc-scene` | Save the current scene file incrementing the file name with a numerical, incrementing, suffix. | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`-n, --number` | If specified, replace the numeric suffix at the end of the scene file name with the given number (or add it if one doesn't exist). | int | Yes<br>`-i, --increment` | If specified, increment the scene file suffix by the given amount (default = 1). | int | Yes |
| `product-list` | List the product information for objects in the current scene | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`-t, --target-file` | File to write output information to | str | Yes<br>`-n, --node-context` | If specified, present the output in the context of nodes. Otherwise the default is to present output in the context of products. | flag (boolean) | Yes<br>`-s, --selected-only` | Only identify products for the currently selected node. | flag (boolean) | Yes |
| `batch-render` | Given a pattern of scene files, load each one and render it to the target location (default is studio-iray) with the given output file pattern | Arg Name | Description | Type | Optional<br>--- | --- | --- | ---<br>`-s, --scene-files` | Pattern of scene files to load, including glob support. See README for examples | str | Yes<br>`-o, --output-path` | Path to directory where output files are to be written. If not specified, use the location of the scene file. | str | Yes<br>`-t, --target` | Target of the render. Allowed values are local-to-file, local-to-window, or iray-server-bridge | str | Yes<br>`-r, --resolution` | Resolution to render, overriding the resolution of the scene file, in WxH format (e.g. 1920x1080) | str | Yes<br>`-c, --cameras` | Cameras to render for. Can be one of 'all_visible', 'viewport', or a pattern that maps to one or more visible cameras | str | Yes<br>`-j, --job-name-pattern` | Pattern to use for naming job names or render output files. In the pattern, %s refers to the scene name, %c the camera name, %f the frame number, and %r the render count | str | Yes<br>`-f, --frames` | Frames to render. Comma-separated list of frames that can include ranges and range patterns. See README for full detail and example. | str | Yes<br>`--iray-server` | For target iray-server the IP address or hostname of the iRay server/master to use. | str | Yes<br>`--iray-protocol` | Sets the http protocol to use. Allowed values are http or https. | str | Yes<br>`--iray-port` | For target iray-server the TCP port of the iRay server/master. | str | Yes<br>`--iray-user` | For target iray-server the username to connect to the iRay server/master. Must be specified here or in config file specified by --iray-config-file | str | Yes<br>`--iray-password` | For target iray-server the password to connect to the iRay server/master. Must be specified here or in config file specified by --iray-config-file | str | Yes<br>`--iray-config-file` | For target iray-server the configuration options file to use for iRay server/master configuration. Values in this file can be overridden by additional command line arguments | str | Yes |
